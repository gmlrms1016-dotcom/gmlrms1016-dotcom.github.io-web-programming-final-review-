6장 인공지능과 딥러닝6장

① AI의 역사 — 기호주의 ↔ 연결주의

'인공지능' 용어를 처음 제시한 사람은 존 매카시(1956 다트머스 회의). 튜링은 '튜링 테스트'를 제안했습니다.

구분설명
기호주의규칙(지식)을 사람이 직접 입력 → 규칙 기반
연결주의인공신경망(뇌의 신경망 흉내)으로 학습 → 딥러닝으로 발전
초기엔 기호주의, 딥러닝은 연결주의의 발전. "AI 전체 = 기호주의도 포함"이므로 '데이터 기반 학습'은 머신러닝(ML)의 설명입니다.

② 머신러닝 = 데이터로 규칙을 자동 생성

전통 프로그래밍은 사람이 규칙을 작성하지만, 머신러닝은 데이터로 규칙을 자동 생성합니다.

머신러닝 수행 과정 (순서 시험 단골!) : 수집 → 전처리 → 학습 → 평가 → 시각화

용어뜻
표본 = 인스턴스 = 데이터 포인트모두 같은 의미 (데이터 한 개)
레이블(label)정답값 (위 용어들과 의미가 다름)
분류(Classification)남/여 같은 '유형'을 맞힘 (유형 수 = 2)
회귀(Regression)연속적인 '값'을 예측

③ 학습 방법 3가지 (가장 중요!)

종류특징예시
지도학습레이블(정답) 있음 — 분류·회귀선형모델, KNN
비지도학습레이블 없음 — 비슷한 것끼리 묶음군집화(K-평균), 차원축소
강화학습보상(reward)을 최대화하도록 학습게임 AI, 로봇 제어
KNN·선형모델 = 지도학습, K-평균·차원축소 = 비지도학습, 보상 최대화 = 강화학습. 자주 헷갈리니 꼭 구분!

④ 퍼셉트론 · 신경망 학습

용어뜻
입력(Input)학습 데이터로 주어짐
가중치(Weight)입력의 중요도 — 학습으로 구해지는 결과값
편향(Bias)활성화 여부 기준값 — 학습으로 구해지는 결과값
출력(Output)가중치·편향을 이용해 계산
활성화 함수ReLU, 시그모이드(Sigmoid)
손실=비용=목적 함수오차(목표값) — 활성화 함수 아님!
오차역전파오차를 출력층 → 입력층으로 거꾸로 전달
경사하강법손실 최소 지점을 찾아가는 방법 (최소화 방향 학습)

⑤ 딥러닝 프레임워크 · 도구

도구특징 / 만든 곳
텐서플로(TensorFlow)구글
케라스(Keras)프랑소아 숄레 — 고수준 API
파이토치(PyTorch)테슬라 오토파일럿, 우버 Pyro 사용

⑥ 괄호채우기 정답

  1. 특이점(Singularity) — AI가 인간 지능을 넘어서는 순간
  2. GAN(생성적 적대 신경망) — 진짜 같은 가짜를 생성
  3. 딥페이크(Deepfake) — 원본 위에 다른 영상을 합성
  4. 연결주의(Connectionism) — 인간의 신경망을 흉내
  5. 학습(훈련) 데이터(Training data) — 모델 학습용 데이터(≠ test data)
  6. 군집화(Clustering) — 정답 없이 비슷한 특성끼리 묶음(비지도)
  7. 강화학습 — 보상을 통해 학습
  8. 은닉층(Hidden Layer) — 입력층과 출력층 사이의 층
  9. 심층신경망(DNN) — 은닉층이 여러 개인 신경망

7장 컴퓨팅 사고력7장

① 컴퓨팅 사고력이란

전문가만이 아니라 모두가 배워야 할 보편 역량이며, '인간'의 사고방법입니다(컴퓨터의 사고 아님). 코딩 능력'만'을 뜻하지 않습니다.

② 구성 요소 4가지 (시험 핵심!)

요소뜻
문제 분해(Decomposition)복잡한 문제를 작은 문제로 나눔
패턴 인식(Pattern Recognition)유사·반복되는 특징을 찾아 일반화
추상화(Abstraction)핵심만 남기고 불필요한 세부 제거·단순화
알고리즘(Algorithm)해결 절차를 순서대로 표현
'문제 융합'은 구성 요소가 아닙니다(올바른 건 문제 분해). 마인드맵·지하철 노선도는 추상화의 예입니다.

③ 추상화 — 자세히

핵심 요소만 남기고, 중요 정보를 선택하며, 불필요 세부를 제거하고, 공통 특징을 일반화하여 현실 문제를 모델로 표현하는 과정. (예: 지하철 노선도 — 실제 거리·곡선은 생략하고 역 순서만 표현)

④ 알고리즘 표현 방법

방법설명
순서도(흐름도, 플로차트)표준 기호·도형으로 절차를 그림으로 표현
의사코드(Pseudocode)프로그래밍 언어와 비슷한 형식의 글
자연어일반 문장으로 표현
알고리즘은 의사코드'만'이 아니라 순서도·자연어 등 여러 방법으로 표현됩니다. 마인드맵은 하향식(중심 → 세부), 트리(나무) 구조입니다.

⑤ SW 교육 방법 · 도구

언플러그드(Unplugged)컴퓨터(도구) 없이 연필·종이·게임으로 학습
피지컬 컴퓨팅로봇·블록·회로로 직접 만들며 배움
앱 인벤터(App Inventor)블록으로 앱을 만드는 교육 도구
블록리(Blockly)구글이 만든 블록 기반 교육 라이브러리

우리나라 초등학생도 SW교육으로 컴퓨팅 사고력을 배웁니다.

9장 데이터베이스9장

① DB의 특징

통합(중복 최소화), 저장, 공유(동시 접근), 실시간 접근, 계속적 변화, 데이터 독립성, 무결성.

DB의 특징은 중복 '최소화'지 '중복 데이터'가 아닙니다. 잘못된 값 입력 → 무결성 위반.

② 기본 용어 & DB vs DBMS

용어뜻
필드(Field)데이터의 종류(열)
레코드(Record)관련 필드의 모음(행) — '필드 종류'가 아님
DB관련 데이터의 집합(저장소)
DBMSDB를 생성·관리·운영하는 소프트웨어(사용자–DB 인터페이스)

③ 데이터 추상화 3단계 & 스키마·독립성

외부 단계사용자 뷰(View)
개념 단계전체 논리 구조
내부 단계물리적 저장 구조
스키마(Schema)DB 구조·제약조건을 명세한 것
논리적 독립성개념 스키마 변경이 외부 스키마에 영향 없음
물리적 독립성내부(저장구조) 변경이 개념 스키마에 영향 없음

④ DB 모델 3가지

모델특징
계층형트리 구조, 부모–자식 1:N (자식 여러 개 가능)
망형그래프 구조, N:M, 링크로 연결
관계형테이블·키 사용, 포인터 없음

⑤ 관계형 구성요소 & 키

릴레이션(Relation)테이블
튜플(Tuple)행(=관계 사례=인스턴스)
속성(Attribute)열
도메인(Domain)속성이 가질 수 있는 값의 범위
주키(Primary Key)튜플을 식별하는 대표키
후보키(Candidate Key)유일성 + 최소성 만족
외래키(Foreign Key)다른 릴레이션의 기본키를 참조

⑥ 관계 연산

Insert레코드(튜플) 생성·삽입
Delete레코드 삭제
Select(Project)부분집합으로 새 관계 생성 — 'Search'가 아님
조인(Join)공통 속성으로 두 릴레이션 결합
디비전(Division)나누기 연산
교집합(Intersection)공통 튜플 추출
merge는 관계 연산이 아닙니다. 레코드 생성 = insert.

⑦ DBMS 종류 · ODBC · NoSQL

MySQL오라클 소유 오픈소스
SQLite모바일 내장 DB (윈도우에서도 사용 가능)
몽고 DB(MongoDB)문서형 NoSQL, 형식은 JSON(BSON)
ODBC응용프로그램이 각 DB 고유 인터페이스를 몰라도 표준 방식으로 다양한 DB에 접근하게 하는 미들웨어 (DB 종류 독립, 교체 쉬움)
다양한 DB 표준 접근 = ODBC. 문서 DB(몽고)는 비관계형(NoSQL)이지 RDBMS가 아닙니다.

9A (추정)강의 '관계의 특징'은 원자값·순서 무의미·중복 튜플 불가. A만 강의 목록에 없어 정답 추정
10D외래키 = 다른 릴레이션의 기본키 참조(후보키 모음 아님)
11Cmerge는 관계 연산 아님
12B레코드 모음은 '파일/테이블'(데이터베이스 아님)
13A레코드 생성 = insert
14C다양한 DB 표준 접근 = ODBC
15DSQLite는 윈도우에서도 사용 가능
16A가장 먼저 = 비즈니스 요구 정의
17A문서 DB는 비관계형(NoSQL)이지 RDBMS 아님
18B몽고 DB만 NoSQL(나머지는 관계형)
19D몽고 DB 문서 형식 = JSON(BSON)
20CDB↔DB가 맞음. BSON은 데이터 형식이라 대응 틀림

⑧ 괄호채우기 정답

  1. 데이터(Data) — 가공되지 않은 사실(값)
  2. 데이터베이스 — 관련 데이터의 집합
  3. 필드(Field) — 의미 있는 자료의 단위(열)
  4. 스키마(Schema) — DB 구조의 구체적 기술
  5. 외부(External) 단계 — 사용자 뷰 단계
  6. 관계 사례(=인스턴스) — 한 행(튜플)
  7. 속성(Attribute) / 튜플(Tuple) — 열 / 행
  8. 데이터 마이닝 — 대량 데이터에서 패턴·규칙 분석·추출
  9. 문서 데이터베이스(NoSQL/몽고DB) — JSON 유사 형식의 비관계형 DB
  10. 검색(조회, Select) — SELECT * FROM … 으로 모든 내용 조회

⑨ 주관식 모범답안

  • 1. DB 특징 — 통합(중복 최소), 저장, 공유(동시 접근), 실시간 접근, 계속적 변화, 데이터 독립성, 무결성
  • 2. DB vs DBMS — DB는 관련 데이터의 집합(저장소), DBMS는 DB를 생성·관리·운영하는 소프트웨어(사용자–DB 인터페이스)
  • 3. 추상화 3단계 — 외부(사용자 뷰) / 개념(전체 논리 구조) / 내부(물리적 저장)
  • 4. 스키마·독립성 — 스키마: DB 구조·제약조건 명세 / 논리적 독립성=개념 스키마 변경이 외부 스키마에 영향 없음 / 물리적 독립성=내부 변경이 개념 스키마에 영향 없음
  • 5. DB 모델 3가지 — 계층형(트리, 1:N) / 망형(그래프, N:M, 링크) / 관계형(테이블·키, 포인터 없음)
  • 6. 관계형 구성요소 — 릴레이션(테이블), 튜플(행), 속성(열), 도메인(속성이 가질 수 있는 값의 범위)
  • 7. 키 — 주키: 튜플 식별 대표키 / 후보키: 유일성+최소성 / 외래키: 다른 릴레이션의 기본키 참조
  • 8. 추가 연산 — 조인(공통 속성으로 결합), 디비전(나누기), 교집합(공통 튜플 추출)
  • 9. SQLite dbtest 예시 — 아래 SQL 참고
  • 10. ODBC — Open Database Connectivity. 응용프로그램이 각 DB 고유 인터페이스를 몰라도 표준 방식으로 다양한 DB에 접근하게 하는 미들웨어. 장점: DB 종류 독립, 호환성·이식성 향상, DB 교체 쉬움

① 네트워크 & OSI 7계층

네트워크는 서로 다른 시스템 간에도 정보를 교환합니다(동일 시스템에만 아님).

OSI 7계층 (ISO 표준) : 물리 → 데이터링크 → 네트워크 → 전송 → 세션 → 표현 → 응용

계층역할
데이터링크인접 노드 간 전송
전송발신지 → 목적지 실제 전송
표현표현 방식 관리·암호화·압축
세션응용 계층에 대응(네트워크 아님)

② 네트워크 장치

라우터(Router)네트워크 계층에서 IP 보고 패킷 최적 경로 결정·전달, 서로 다른 망 연결
브리지/스위치같은 기관 LAN 세그먼트 연결 (둘 이상 LAN을 한 네트워크로 = 브리지)
게이트웨이(Gateway)서로 다른 프로토콜 망 연결(프로토콜 변환)

③ LAN/WAN & 인터넷 역사

이더넷·FDDI 등은 LAN 표준. WAN은 국가·국가 간 연결. 광섬유는 대역폭이 아주 큼. 인터넷의 시초는 ARPANET(NSFNET 아님). 계층 구조는 의존성을 줄이기 위함.

④ TCP/IP 4계층

네트워크 접속 → 인터넷 → 전송 → 응용. TCP/IP는 응용보다 먼저인 기반 구조입니다.

인터넷 계층IP 주소 지정과 라우팅(경로 배정) 담당 (IP·ICMP·ARP)
전송 계층 — TCP연결형, 신뢰성·흐름·오류 제어
전송 계층 — UDP비연결형, 빠르나 신뢰성 낮음

⑤ 웹·프로토콜·마크업

HTTP웹 문서 전송 규약 (웹 서버가 요청에 응답)
FTPFile Transfer Protocol (파일 전송) — 'Folder' 아님
SMTP이메일 통신 규약
Telnet원격 접속
WWW동영상·기타 파일도 제공
HTML / XMLHyperText / eXtensible Markup Language (eXtreme 아님)
ASP서버에서 처리 (브라우저가 해독 안 함)

⑧ 주관식 모범답안

  • 1. OSI 7계층 — ISO가 정한 통신 표준(물리·데이터링크·네트워크·전송·세션·표현·응용). 필요성: 계층별로 나눠 독립적 표준화 → 호환성 확보, 유지보수·문제해결 용이, 이기종 간 통신 가능
  • 2. 게이트웨이/라우터 — 게이트웨이: 서로 다른 프로토콜 망 연결(프로토콜 변환) / 라우터: 네트워크 계층에서 IP 보고 패킷 최적 경로 결정·전달
  • 3. 인터넷 응용 서비스 — WWW(웹), 전자우편(E-mail), FTP(파일전송), Telnet(원격접속), 메신저, 클라우드 등
  • 4. TCP/IP 4계층 — 네트워크 접속 / 인터넷 / 전송 / 응용. 인터넷 계층: IP 주소 지정과 라우팅 담당(IP·ICMP·ARP)
  • 5. 전송 계층 2개 — TCP(연결형, 신뢰성·흐름·오류 제어) / UDP(비연결형, 빠르나 신뢰성 낮음)
  • 6. 차세대 IP — IPv6. 가장 중요한 이유: IPv4 주소 고갈 해결(128비트로 주소 공간 대폭 확장, 보안 강화)
  • 7. IP 클래스 — A(대형망)·B(중형망)·C(소형망)·D(멀티캐스트)·E(예약/연구용)
  • 8. IPv6 형태 — 128비트, 16비트씩 8그룹, 콜론(:)으로 구분, 16진수 표기 (예: 2001:0db8:85a3::8a2e:0370:7334)
  • 9. URL 분석 — 프로토콜://호스트(도메인):포트/경로/파일?질의 구조 (예: https://www.dongyang.ac.kr/path/index.html → https=프로토콜, dongyang.ac.kr=호스트, /path/=경로, index.html=파일)
